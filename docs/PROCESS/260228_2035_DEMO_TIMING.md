**ANIMATION SPECIFICATIONS**

1. **Agent Terminal entries**: Fade-and-slide in from below (translateY 8px → 0, opacity 0 → 1, 200ms ease-out). When a red/critical event type appears (SEVERITY_DELTA, HAZARD_DETECTED), that entire log entry gets a faint red background highlight that fades over 1s. The blinking cursor at bottom is great — keep it.

2. **Severity badge transition**: When severity changes (e.g., HIGH → CRITICAL), the badge should scale up briefly (1.0 → 1.1 → 1.0 over 400ms) with a red glow pulse that radiates outward and fades over 600ms. The entire Case File panel border should flash red for 200ms then fade back. This is the single most important animation in the demo.

3. **Deployment cards**: Slide in from the right (translateX 20px → 0, opacity 0 → 1, 300ms) one at a time, sequentially, as each dispatch event arrives from Supabase. NOT all at once. When a card transitions from RECOMMENDED to CONFIRMED, the left accent border should animate from grey to the accent color as a wipe-down effect over 300ms. The "4 UNITS" counter should animate up: 0 → 2 → 3 → 4, with a brief number flip on each increment.

4. **Transcript messages**: Each new message fades in (200ms). The entity extraction tags ("market and 5th", "trapped", "child") should appear 300ms AFTER the message text with a pop animation (scale 0.8 → 1.0, opacity 0 → 1). This creates the visual of the system reading and extracting in real time.

5. **CCTV hazard detection**: When HAZARD_DETECTED fires, the bounding box transitions from dashed grey to solid bright red over 200ms. A detection label fades in above/below the box: "ENGINE FIRE · 0.99" with a semi-transparent red background pill. The CCTV panel border pulses red twice (200ms on, 200ms off, 200ms on) then settles. After the moment passes the label stays but the pulse stops.

6. **New caller tab**: When Caller 2 (ZH) arrives, the tab slides in from the right and gets a pulsing notification dot (small green/yellow circle) for 3 seconds. In the transcript panel, insert a subtle divider: "— NEW CALLER CONNECTED · ZH —" with a timestamp, dimmer text color than regular messages.

7. **Approve button — three states**:
   - BEFORE gate: `opacity: 0.3`, muted grey border, grey text, `pointer-events: none`. Nearly invisible.
   - WAITING (gate reached): Animate in with scale-up (0.95 → 1.0). Slow breathing glow — border and background opacity cycling 0.7 → 1.0 over 2s, ease-in-out, infinite. Text: "APPROVE INITIAL RESPONSE". Must magnetically pull the eye.
   - APPROVED (clicked): Brief bright flash (100ms white overlay), then settle to solid green with "✓ RESPONSE APPROVED". After 3 seconds, fade to a muted confirmed state (less bright, no glow) so it doesn't compete with later events.

8. **Active Hazard badges**: Appear ONE AT A TIME as evidence arrives, not all at once. TRAPPED OCCUPANT after Caller 1 intake. CHILD PRESENT after Caller 2 correlation. ENGINE FIRE after vision detection. Each badge pops in (scale 0.8 → 1.0, 200ms).

9. **Case File incident title**: Should evolve. Start as "INCOMING CALL" or "UNCLASSIFIED" → update to "VEHICLE COLLISION" after first triage → update to "VEHICLE COLLISION + FIRE" after vision. Each change should have a brief crossfade (old text fades out, new text fades in, 300ms).
