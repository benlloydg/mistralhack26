# DISPATCH — After-Action Report Page Design PRD

## Overview

After the demo completes (explosion detected, scene secured, zero casualties confirmed), the presenter clicks a `[GENERATE REPORT]` button in the Response Actions column. The screen transitions to a full-page after-action report at a shareable URL: `dispatch.app/cases/TN-2026-00417`.

This page is a read-only, static render of the frozen case state. No Realtime subscriptions, no animations, no live data. It queries the final state of all Supabase tables for the given case_id and renders a clean, professional, printable report.

This page serves three purposes:
1. **Demo closer.** The hard cut from explosion → report is the mic drop moment.
2. **Judge artifact.** The URL goes on the hackathon submission page. Judges can open it on their laptops and examine the full audit trail at their own pace.
3. **Portfolio piece.** This is what you screenshot, share on Twitter, and put in your deck.

---

## Trigger

A new action card appears in the Response Actions column after the system logs `SCENE SECURED`:

```
┌────────────────────────────────────────────────┐
│  📋 AFTER-ACTION REPORT              [GENERATE]│
└────────────────────────────────────────────────┘
```

When clicked:
1. `POST /cases/{case_id}/report` — backend generates the case summary via Mistral Large if not already cached
2. Frontend transitions to the report page (full-screen, replaces the dashboard)
3. URL updates to `/cases/TN-2026-00417` (shareable, bookmarkable)

The transition should feel deliberate — a 500ms fade-to-black, then the report fades in. Like closing one chapter and opening the final page. Not a jarring page navigation.

---

## Page structure

The report is a single scrollable page. No sidebar, no navigation, no interactive elements except a "Back to Dashboard" link and a print/export button. It should feel like a government incident report crossed with an Apple keynote summary — dense with information but beautifully typeset.

Dark background (consistent with the dashboard). Clean monospace + sans-serif type pairing. Wide margins. Generous whitespace between sections.

## Technical implementation

### Data source
The report page queries these Supabase tables (read-only, no subscriptions):
- `incident_state` — final case state, severity, hazards, outcome
- `transcripts` — all committed transcripts with timestamps and translations
- `agent_logs` — full agent activity log with model names
- `dispatches` — all response actions with status and authorization method
- `vision_detections` — frame analysis results with extracted frame images

### Route
```
/cases/[case_id]
```
Standard Next.js dynamic route. Server-side rendered or static — no client-side interactivity needed except the print button.

### Frame storage
During the demo, when the vision agent extracts and analyzes frames, save the frame images:
```python
frame_path = f"assets/frames/{case_id}_t{timestamp}s.jpg"
cv2.imwrite(frame_path, frame)
```
The report page loads these as images. If frames weren't saved, show placeholder boxes with the detection metadata.

### Executive summary generation
```python
async def generate_executive_summary(case_id: str) -> str:
    state = await get_case_state(case_id)
    logs = await get_agent_logs(case_id)
    transcripts = await get_transcripts(case_id)
    
    prompt = f"""Generate a 3-4 sentence executive summary of this incident.
    
    Case: {json.dumps(state)}
    Timeline events: {len(logs)}
    Audio sources: {len(set(t['language'] for t in transcripts))} languages
    Key facts: {state['hazard_flags']} + {state['injury_flags']}
    Outcome: {state['status']}
    
    Be factual, concise, and emphasize the cross-modal evidence fusion
    and the evacuation outcome. This reads like an official incident report."""
    
    response = await mistral.chat.complete(
        model="mistral-large-latest",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3,
    )
    return response.choices[0].message.content.strip()
```

### Print styling
Add a `@media print` stylesheet that:
- Switches to white background, black text
- Hides the Back to Dashboard link
- Ensures the convergence timeline renders cleanly
- Adds page breaks before major sections
- Includes the URL in the footer for attribution

## Transition from dashboard

When the presenter clicks `[GENERATE REPORT]`:

1. The action card shows a brief loading state: `GENERATING...` with a spinner (1-2 seconds while the executive summary generates, or instant if pre-cached)
2. The entire dashboard fades to black (500ms)
3. The report page fades in from the top (500ms)
4. The report auto-scrolls very slowly from top to bottom over ~10 seconds if no user interaction (optional — good for the demo video recording, the audience sees the full report scroll by)

For the 2-minute submission video, the report should be on screen for approximately 5-8 seconds. Enough for the audience to read "ZERO CASUALTIES" and see the timeline structure.
