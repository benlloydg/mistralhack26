# DISPATCH — 55-Second Technical Demo Script v3

## Core concept shift (v2 → v3)

**No phone calls. No callers. No 911.**

DISPATCH passively monitors a scene via CCTV video + ambient microphone. There are no calls, no callers, no phone lines. The system listens to the environment — bystander voices, ambient sounds, visual changes — and builds intelligence from raw sensory input.

One continuous audio stream from one microphone. ElevenLabs Scribe v2 Realtime segments speakers, detects languages, and commits transcripts. The agents extract intelligence from each committed segment. The vision pipeline independently analyzes video frames. Evidence fusion merges both modalities into a shared case state.

The system doesn't wait for humans to call in. It's already listening. It's already watching. When something happens, it's the first to know.

---

## The narrative arc in one sentence

The system detects pre-crash panic in four languages, confirms the collision via audio and vision simultaneously, builds a case from three bystander reports across three languages, catches a fire that only one bystander mentions, autonomously broadcasts evacuation warnings in all detected languages, and confirms zero casualties after the explosion.

---

## Demo arc — 5 state transitions in 55 seconds

```
MONITORING → ACTIVE → HIGH → CRITICAL → SCENE SECURED
```

Each transition is visible in the case file. The system goes from passive listening to situational awareness to active response to autonomous action to verified outcome.

## Scene Audio Panel Design

Single scrolling transcript. No tabs. No entity extraction tags (those appear in the Case File after agent processing). Format per entry:

```
[timestamp]                                      LANG → EN
  "Original language text"
  English translation
```

The evacuation outbound collapses to one compact entry:

```
[timestamp] ⚠ DISPATCH → SCENE                 EVACUATION
  🔊 ES ✓ · ZH ✓ · FR ✓
```

A live waveform/level meter at the top of the panel driven by `AnalyserNode` on the video's audio context. Spikes when someone speaks, flat during silence, maxes at crash and explosion.

Partial transcripts from Scribe appear as a shimmering/typing line at the bottom:

```
▸ DETECTING: "后座有个小..." ▌
```

This resolves into the committed transcript when Scribe finalizes.

## Key technical beats for judges

1. **Pre-incident awareness.** The system detects rising tension before the crash. It doesn't just react — it anticipates.

2. **Simultaneous audio + visual impact detection.** The collision is confirmed by two independent modalities at the same moment.

3. **Multilingual ambient transcription.** Four languages from one microphone. No phone calls, no routing, no language selection. Scribe figures it out.

4. **Cross-modal evidence fusion.** Fire detected by vision (camera) AND audio (French bystander) independently. The system logs the corroboration explicitly.

5. **Vision-first detection.** The camera detects smoke before any bystander mentions it. The system sees what humans haven't reported yet.

6. **Autonomous evacuation.** The system broadcasts warnings without waiting for human approval because fire + trapped person + child = too urgent to wait. But unit dispatches still require human authorization. The system knows the difference.

7. **Verified outcome.** After the explosion, the system confirms zero casualties. It closed the loop on its own actions.

8. **Partial transcript shimmer.** Live Scribe partial events visible in the UI prove real-time processing, not pre-recorded playback.

9. **After-action report.** A shareable URL with full timeline, source convergence visualization, agent utilization stats, and executive summary.
